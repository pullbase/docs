Pullbase ships as a Docker image. This guide walks you through running the central server with PostgreSQL, bootstrapping an admin account, and confirming the system is healthy.

## Prerequisites

- Docker 24.0 or newer with the Compose plugin (`docker compose`)
- Git (to clone the sample configuration repository)
- Open ports 8080 (Pullbase API) and 5432 (PostgreSQL) on your host
- A Unix-like shell (macOS, Linux, or WSL)

<Note>
Pullbase runs on HTTP internally. For production deployments, place it behind a reverse proxy (NGINX, Traefik, Caddy) that terminates TLS.
</Note>

## Create a working directory

```bash
mkdir pullbase-quickstart && cd pullbase-quickstart
git clone https://github.com/pullbaseio/pullbase-config-examples.git configs
```

The `configs` repository contains sample environment definitions you can deploy while exploring Pullbase.

## Compose file

Create `docker-compose.yml` with the minimal services required to run Pullbase locally:

```yaml docker-compose.yml
version: "3.9"
services:
  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: pullbaseuser
      POSTGRES_PASSWORD: pullbasepass
      POSTGRES_DB: pullbasedb
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U pullbaseuser -d pullbasedb"]
      interval: 10s
      timeout: 5s
      retries: 5

  pullbase:
    image: pullbaseio/pullbase:latest
    depends_on:
      db:
        condition: service_healthy
    ports:
      - "8080:8080"
    environment:
      # Database configuration
      PULLBASE_DB_HOST: db
      PULLBASE_DB_PORT: 5432
      PULLBASE_DB_USER: pullbaseuser
      PULLBASE_DB_PASSWORD: pullbasepass
      PULLBASE_DB_NAME: pullbasedb
      PULLBASE_DB_SSLMODE: disable
      # Server configuration
      PULLBASE_SERVER_PORT: 8080
      PULLBASE_SERVER_HOST: 0.0.0.0
      # Authentication
      PULLBASE_JWT_SECRET: change-this-super-secret-key-please
      PULLBASE_JWT_EXPIRY_HOURS: 24
      # Webhook security
      PULLBASE_WEBHOOK_SECRET_KEY: d5aaaf2f8d3f514a9e649416f3e62f007f878a049e5fc277d11c0003d1e24ff4
      # Git integration (disable for quickstart)
      PULLBASE_GIT_ENABLED: "false"

volumes:
  postgres_data:
```

<Info>
The sample configuration disables Git integration for the quickstart. When you create environments via the API or UI, you'll specify repository details per-environment. For private repositories, configure a GitHub App as described in the GitHub App Integration guide.
</Info>

## Start Pullbase

<Steps>
<Step title="Launch the stack">
  ```bash
  docker compose up -d
  ```

  <Check>
  Verify the containers are healthy:
  
  ```bash
  docker compose ps
  ```
  </Check>
</Step>

<Step title="Check health">
  ```bash
  curl http://localhost:8080/api/v1/healthz
  ```
  You should see `{"status":"healthy","service":"pullbase-server"}`.
</Step>

<Step title="Bootstrap the first admin">
  Pullbase generates a one-time secret at `bootstrap-admin-secret.txt` inside the config directory.
  
  ```bash
  docker compose exec pullbase cat /app/config/bootstrap-admin-secret.txt
  ```
  
  Copy the secret (do not include trailing whitespace) and use the bundled CLI from inside the running container:
  
  ```bash
  docker compose exec pullbase pullbasectl auth bootstrap-admin \
    --server-url http://localhost:8080 \
    --bootstrap-secret-file /app/config/bootstrap-admin-secret.txt \
    --username admin_user \
    --password 'ChangeMeNow123!'
  ```
  
  The command returns a short-lived JWT you can use to sign in. Pullbase removes the bootstrap secret file automatically after a successful bootstrap.
</Step>

<Step title="Open the web UI">
  Visit `http://localhost:8080` and sign in with the credentials you just created.

  <Check>
  You should see the dashboard. From here you can create environments and register servers.
  </Check>
</Step>
</Steps>
