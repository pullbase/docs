Pullbase delivers an Argo CDâ€“inspired GitOps workflow for traditional servers. Operators define the desired state of packages, services, and files in Git, while the Pullbase agent continuously reconciles that state on each node.

## Why Pullbase

- **Git as the source of truth:** Track every configuration change with version control.
- **Pull-based reconciliation:** Agents fetch desired state, eliminating firewall headaches and reducing blast radius.
- **Environment-driven design:** Group servers, inherit configuration from environments, and roll back with confidence.
- **Unified toolchain:** Central server, UI, CLI, and API work together for onboarding, operations, and automation.
